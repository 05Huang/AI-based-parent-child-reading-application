# 快速启动指南

## 5分钟快速上手

### 第1步：启动后端 (2分钟)

```bash
# 1. 打开PowerShell终端
# 2. 进入后端项目目录
cd D:\AIParentChildReadingApp\qz-sns-app\qz-sns-web

# 3. 启动Spring Boot应用
mvn spring-boot:run

# 等待启动完成，看到类似输出:
# Started QZServerApp in 15.234 seconds
```

**验证后端启动**:
- 打开浏览器访问: http://localhost:8888/api/user/page?current=1&size=10
- 应该能看到JSON返回（可能为空或包含用户数据）

### 第2步：启动前端 (2分钟)

```bash
# 1. 打开新的PowerShell终端
# 2. 进入前端项目目录
cd "D:\社团组织\社团组织\大创-阅桥亲子阅读APP\Houtaiguanli"

# 3. 安装依赖（如果还没安装）
npm install

# 4. 启动开发服务器
npm run dev

# 等待启动完成，看到输出:
# ➜  Local:   http://localhost:5173/
```

### 第3步：访问应用 (1分钟)

1. 打开浏览器访问: **http://localhost:5173**
2. 导航到用户管理页面
3. 查看用户列表

## 功能演示

### 查看用户列表

```
1. 打开应用首页
2. 点击左侧导航菜单中的"用户管理"
3. 可以看到用户列表，包含以下信息：
   - 用户名
   - 手机号
   - 邮箱
   - 角色（家长/孩子）
   - 状态（正常/禁用）
   - 创建时间
```

### 新增用户

```
1. 点击"新增用户"按钮
2. 填写用户信息：
   - 用户名: 任意字符（2-20位）
   - 昵称: 任意字符（2-20位）
   - 角色: 选择"家长"或"孩子"
   - 手机号: 格式 1[3-9]xxxxxxxx
   - 邮箱: 有效的邮箱地址
   - 密码: 6-20位
3. 点击"确定"按钮
4. 看到"添加成功"提示
5. 用户出现在列表中
```

### 编辑用户

```
1. 在用户列表中找到要编辑的用户
2. 点击该用户行的"编辑"按钮
3. 修改用户信息（不能修改用户名）
4. 点击"确定"按钮
5. 看到"编辑成功"提示
```

### 禁用/启用用户

```
1. 在用户列表中找到要禁用的用户
2. 点击该用户行的"禁用"按钮
3. 确认操作
4. 用户状态变为"禁用"
5. 再次点击该按钮可以"启用"用户
```

### 删除用户

```
1. 在用户列表中找到要删除的用户
2. 点击该用户行的"删除"按钮
3. 确认删除操作
4. 看到"删除成功"提示
5. 用户从列表中移除
```

### 搜索用户

```
1. 在搜索区域输入搜索条件：
   - 用户名: 输入用户名搜索
   - 角色: 选择"家长"或"孩子"
   - 状态: 选择"正常"或"禁用"
2. 点击"搜索"按钮
3. 列表显示符合条件的用户

搜索技巧：
- 可以组合多个条件进行精确搜索
- 点击"重置"清除所有条件
```

## 故障排除

### 问题：前端无法连接后端

**症状**:
- 打开应用时看到加载错误
- 控制台显示 `Failed to fetch` 或网络错误

**解决**:
```bash
# 1. 检查后端是否运行
curl http://localhost:8888/api/user/page?current=1&size=10

# 2. 如果返回JSON，说明后端正常
# 3. 清除浏览器缓存并刷新页面
# 4. 检查.env.local配置是否正确
```

### 问题：页面加载缓慢

**解决**:
- 这可能是第一次访问或数据量较大
- 刷新页面重试
- 检查网络连接
- 在浏览器开发工具中查看Network标签

### 问题：操作后没有反应

**解决**:
1. 打开浏览器开发者工具 (F12)
2. 查看 Console 标签是否有错误信息
3. 查看 Network 标签中的API请求是否成功
4. 检查后端应用日志

## 开发工具建议

### 浏览器开发工具

```
快捷键: F12 或 Ctrl+Shift+I

关键标签页:
- Console: 查看错误信息和日志
- Network: 监控API请求
- Application: 查看localStorage和sessionStorage
- DevTools: Vue开发工具
```

### API测试工具

推荐使用 Postman 或 Insomnia 测试API:

```
1. 下载 Postman: https://www.postman.com/downloads/
2. 创建新Request
3. 设置URL: http://localhost:8888/api/user/page
4. 设置参数:
   - current: 1
   - size: 10
5. 点击Send查看返回结果
```
