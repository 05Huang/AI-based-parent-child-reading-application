📖 项目简介
-----------
本工具专为亲子阅读APP开发的测试数据生成系统，用于解决测试阶段素材不足的问题。

主要功能：
✓ 从开源平台（Unsplash、Pexels）爬取高质量图片和视频
✓ 严格按照项目规范上传到MinIO对象存储
✓ 自动生成符合业务逻辑的测试文章内容
✓ 生成推荐系统所需的完整数据字段
✓ 批量插入MySQL数据库


===============================================================================

🎯 快速开始（5分钟）

⚙️ 环境要求
-----------

必需：
  ✓ Python 3.7+
  ✓ MinIO服务 (http://127.0.0.1:9000)
  ✓ MySQL数据库 (qz_sns)
  ✓ 网络连接（用于下载素材）

可选：
  ○ Unsplash API密钥（提高图片获取成功率）
  ○ Pexels API密钥（提高图片和视频获取成功率）

🔧 安装步骤
-----------

1. 确保环境已准备
   □ MinIO已启动
   □ MySQL已启动并导入数据库表结构
   □ Python已安装

2. 运行环境检查

3. 安装依赖包

💡 使用建议
-----------

首次使用：
  1. 运行快速测试验证环境
     双击 quick_test.bat
  
  2. 观察日志输出，确认无错误
  
  3. 检查生成结果
     - MinIO: http://127.0.0.1:9000
     - MySQL: SELECT * FROM content ORDER BY id DESC LIMIT 10;

配置优化（推荐）：
  4. 注册并获取API密钥
     - Unsplash: https://unsplash.com/developers
       → 注册应用后复制 Access Key（不是Secret Key）
     - Pexels: https://www.pexels.com/api/
       → 注册账号后复制 API Key
  
  5. 创建配置文件
     复制 config.example.json 为 config.json
     填入API密钥

批量生成：
  6. 运行完整版本生成大量数据
     双击 run_generate_test_data.bat
     
  7. 建议数量
     - 文章: 50-100篇
     - 视频: 10-20个

🗂️ 文件存储路径
----------------

MinIO存储结构：
  qz-sns/
    ├─ image/article/{id}/
    │   ├─ article_cover.jpg      (封面图)
    │   ├─ article_image_1.jpg    (配图1)
    │   └─ article_image_2.jpg    (配图2)
    │
    └─ video/resource/{id}/
        ├─ video_cover.jpg        (视频封面)
        └─ video.mp4              (视频文件)

数据库表：
  - content: 主内容表（type: 1图文, 2视频）
  - content_image: 文章图片关联表
  - content_category: 内容分类表

⚠️ 注意事项
-----------

1. 版权说明
   - 使用的图片和视频来自免费开源平台
   - 仅用于测试和开发目的
   - 遵守各API平台的使用条款

2. API限制
   - Unsplash: 免费50请求/小时
   - Pexels: 免费200请求/小时
   - 建议配置API密钥避免限流

3. 性能考虑
   - 视频文件较大（通常10-50MB）
   - 下载和上传需要时间
   - 建议在网络良好时运行

4. 数据库
   - 生成的ID会自动递增
   - 不会覆盖已有数据
   - 建议在测试数据库中使用

🐛 常见问题
-----------

Q: 提示"MinIO连接失败"？
A: 检查MinIO是否启动: http://127.0.0.1:9000
   确认访问密钥: root/12345678

Q: 提示"MySQL连接失败"？
A: 检查MySQL服务是否启动
   确认用户名密码: root/123456
   确认数据库已创建: qz_sns

Q: 图片/视频获取失败？
A: 可能是网络问题或API限流
   建议配置API密钥
   可以减少生成数量重试

Q: 视频下载很慢？
A: 视频文件通常较大，这是正常现象
   可以先只生成文章（视频数量设为0）

Q: 如何清空测试数据？
A: 执行SQL: 
   DELETE FROM content_image WHERE content_id > 原有最大ID;
   DELETE FROM content WHERE id > 原有最大ID;
