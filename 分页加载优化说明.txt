                        分页加载功能优化说明
===============================================================================

问题描述
--------
文章页面和视频页面只显示前30条数据，无法继续加载更多内容。


优化内容
--------

1. ✅ 添加滚动触发阈值
   - 在 scroll-view 中添加 lower-threshold="100"
   - 距离底部100px时就开始触发加载，提前预加载
   - 优化用户体验，避免滚动到最底部才加载

2. ✅ 修复分页逻辑
   - 在加载更多时，先递增 currentPage，再调用加载方法
   - 在 loadArticles/loadVideos 中不再覆盖 currentPage
   - 确保页码正确递增：1 → 2 → 3 → ...

3. ✅ 增强日志输出
   - 记录每次加载的详细信息：
     * 请求页码
     * 是否刷新模式
     * 收到的数据条数
     * 当前总数据量
     * 是否还有更多
   - 帮助快速定位问题

4. ✅ 优化加载状态管理
   - 防止重复请求
   - 明确区分刷新和加载更多两种模式
   - 正确处理数据追加


工作原理
--------

初始加载：
  1. 页面加载时，currentPage = 1
  2. 请求第1页数据（20条）
  3. 显示数据，hasMore = true

滚动加载更多：
  1. 滚动距离底部100px时触发
  2. currentPage 递增：1 → 2
  3. 请求第2页数据（20条）
  4. 追加到现有数据：20条 → 40条
  5. 继续滚动，重复步骤1-4

加载完所有数据：
  1. 当 currentPage >= totalPages 时
  2. hasMore = false
  3. 显示"没有更多数据了"
  4. 不再触发加载


配置说明
--------

每页数量：
  const pageSize = ref(20)
  
  可以根据需要调整，比如：
  - 20条：快速加载，流畅体验
  - 30条：减少请求次数
  - 50条：一次加载更多

触发距离：
  lower-threshold="100"
  
  可以调整：
  - 50：快到底部才加载
  - 100：提前一点加载（推荐）
  - 200：更早触发，预加载

常见问题排查
------------

Q: 滚动到底部没有加载更多？
A: 检查以下几点：
   1. 查看控制台，是否有"触发加载更多"日志
   2. 检查 hasMore 是否为 true
   3. 确认 scroll-view 高度设置正确
   4. 检查是否有其他元素遮挡滚动

Q: 显示"没有更多数据了"但数据库还有？
A: 检查：
   1. 后端分页查询是否正确
   2. total 和 pages 数值是否正确
   3. 是否有数据被过滤（status、creator_id等）

Q: 数据重复显示？
A: 检查：
   1. 是否多次触发加载
   2. currentPage 是否正确递增
   3. 刷新和加载更多的逻辑是否混淆

Q: 加载很慢？
A: 优化建议：
   1. 增加 pageSize（比如改为30或50）
   2. 检查网络状态
   3. 优化后端查询性能
   4. 考虑添加缓存


测试建议
--------

1. 测试初始加载
   - 刷新页面，应该显示第1页数据

2. 测试加载更多
   - 滚动到底部
   - 观察是否自动加载下一页
   - 检查数据是否追加（不是替换）

3. 测试加载完所有数据
   - 继续滚动直到没有更多数据
   - 应该显示"没有更多数据了"
   - 不应该再发起请求

4. 测试下拉刷新
   - 下拉列表
   - 应该重置为第1页
   - 之前加载的数据被清空

5. 测试快速滚动
   - 快速滚动到底部
   - 不应该发起多次重复请求
   - loading 状态应该正确管理


性能优化
--------

如果数据量特别大（1000+条），建议：

1. 虚拟列表
   - 只渲染可见区域的元素
   - 减少DOM节点数量

2. 懒加载图片
   - 图片进入视口时才加载
   - 减少初始加载时间

3. 骨架屏
   - 加载时显示占位内容
   - 提升用户体验

4. 缓存策略
   - 缓存已加载的数据
   - 减少重复请求
